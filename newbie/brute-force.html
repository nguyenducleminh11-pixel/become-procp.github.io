Cách nó hoạt động

Thay vì tìm công thức toán học phức tạp, ta dựng lên một "vòng lặp" đi qua tất cả các khả năng. Giống như việc thử từng chiếc chìa khóa trong chùm chìa để mở cửa.

Tại sao phải học?

Trong CP, Brute force giúp bạn: 1. Ăn điểm các subtask nhỏ. 2. Làm "đáp án mẫu" để Stress Test cho thuật toán chính.

Mô phỏng: Tìm x sao cho x² = 16 Simulation: Find x such that x² = 16

Thử x = 1 1 ≠ 16
Thử x = 2 4 ≠ 16
Thử x = 3 9 ≠ 16
Thử x = 4 16 = 16!

Duyệt tuần tự: Đơn giản, chính xác, nhưng chậm. Sequential search: Simple, accurate, but slow.

Độ phức tạp Giới hạn N thường gặp Loại bài toán
$O(N!)$ $N \le 10 \sim 12$ Hoán vị, đường đi ngắn nhất qua mọi điểm.
$O(2^N)$ $N \le 20 \sim 25$ Tập con, bài toán cái túi cơ bản.
$O(N^3)$ $N \le 500$ Duyệt 3 vòng lặp lồng nhau.
$O(N^2)$ $N \le 5000$ Cặp phần tử, bảng phương án.

Bài toán mẫu: Triple Sum Zero

Cho mảng N số. Đếm xem có bao nhiêu bộ 3 số $(i, j, k)$ mà $A[i] + A[j] + A[k] = 0$. Given an array of N numbers. Count triplets $(i, j, k)$ such that $A[i] + A[j] + A[k] = 0$.

solution.cpp 

#include <iostream>
#include <vector>

int countTriplets(std::vector<int>& a) {
    int count = 0;
    int n = a.size();
    
    // Ba vòng lặp lồng nhau đại diện cho Brute Force O(N^3)
    for (int i = 0; i < n; i++) {
        for (int j = i + 1; j < n; j++) {
            for (int k = j + 1; k < n; k++) {
                if (a[i] + a[j] + a[k] == 0) {
                    count++;
                }
            }
        }
    }
    return count;
}
